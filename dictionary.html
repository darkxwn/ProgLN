Термины языков программирования

1. Общие характеристики языков

1.1 Мультипарадигменный язык

Поддерживает несколько парадигм программирования, таких как процедурная, объектно-ориентированная, функциональная и др.

Примеры: Python, JavaScript, Scala

1.2 Кроссплатформенный язык

Позволяет запускать программы на разных операционных системах без изменения исходного кода.

Примеры: Java, Python, Go

1.3 Эзотерический язык?

Язык, созданный не с целью практического применения, а как эксперимент, шутка, исследование теоретических пределов программирования или способ самовыражения. Часто такие языки экстремально минималистичны, намеренно сложны или необычны по синтаксису.

Примеры:

  • Brainfuck — минималистичный язык с 8 командами, крайне трудно читается человеком.
  • LOLCODE — использует синтаксис в стиле интернет-сленга ("LOLspeak").
  • Whitespace — код состоит только из пробелов, табов и переводов строк.
  • Malbolge — намеренно запутанный, один из самых сложных языков.
  • Befunge — исполняется в двумерной плоскости, а не линейно.

1.4 Интерпретируемый язык

Программы выполняются интерпретатором построчно, без предварительной компиляции.

Примеры: Python, JavaScript, Ruby

1.5 Компилируемый язык

Исходный код сначала компилируется в машинный код, после чего выполняется.

Примеры: C, C++, Rust

1.6 Статическая типизация

Типы переменных определяются и проверяются во время компиляции.

Примеры: Java, C++, Rust

1.7 Динамическая типизация

Типы переменных определяются во время выполнения.

Примеры: Python, JavaScript, Ruby

1.8 Явная типизация

Программист должен явно указывать тип переменной.

Примеры: C, Java

1.9 Неявная типизация

Тип переменной выводится автоматически (инференция типов).

Примеры: Python, JavaScript, TypeScript (частично)

1.10 Байт-код

Промежуточный код, исполняемый виртуальной машиной (например, JVM или CLR).

Примеры: Java, C#

1.11 Асинхронное программирование

Подход, при котором операции могут выполняться параллельно без блокировки основного потока выполнения.

Примеры: async/await в JavaScript, asyncio в Python

2. Парадигмы программирования

2.1 Процедурная парадигма

Программы строятся из процедур (функций), выполняемых последовательно.

Примеры: C, Pascal

2.2 Объектно-ориентированная парадигма (ООП)

Основана на понятиях классов и объектов. Поддерживает наследование, инкапсуляцию и полиморфизм.

Примеры: Java, C++, Python

2.3 Функциональная парадигма

Функции являются основными единицами. Важно избегание побочных эффектов и неизменяемость данных.

Примеры: Haskell, F#, Scala

2.4 Императивная парадигма

Программист описывает, как выполнить задачу, указывая пошаговые инструкции.

Примеры: C, Python, Java

2.5 Декларативная парадигма

Программист описывает, что нужно получить, а не как это сделать.

Примеры: SQL, HTML, Prolog

2.6 Реактивное программирование

Основано на потоках данных и автоматической реакции на их изменения.

Примеры: RxJS в JavaScript, ReactiveX

2.7 Логическое программирование

Основано на логике и правилах вывода, используется в системах ИИ.

Примеры: Prolog

2.8 Метапрограммирование

Позволяет программе модифицировать свой собственный код во время выполнения или компиляции.

Примеры: Python (decorators), C++ (templates)

2.9 Рефлексия

Способность программы исследовать и изменять свою структуру во время выполнения.

Примеры: Java, C#
